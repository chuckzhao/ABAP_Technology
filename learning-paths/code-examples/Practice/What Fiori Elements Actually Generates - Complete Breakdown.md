## The Generation Process: Runtime Flow

### When You Open the App

```
User enters URL
    ‚Üì
1. Browser loads manifest.json (YOUR file)
    ‚Üì
2. Manifest says: "Use sap.fe.templates.ListReport"
    ‚Üì
3. Fiori Elements framework initializes
    ‚Üì
4. FE reads: "dataSource" from manifest
    ‚Üì
5. FE calls: GET /$metadata (from OData service)
    ‚Üì
6. FE receives: XML metadata with @UI annotations
    ‚Üì
7. FE parses annotations:
   - @UI.lineItem ‚Üí Table columns
   - @UI.selectionField ‚Üí Filter fields
   - @UI.identification ‚Üí Form fields
    ‚Üì
8. FE GENERATES JavaScript controller in memory
    ‚Üì
9. FE GENERATES HTML DOM structure
    ‚Üì
10. FE APPLIES CSS classes
    ‚Üì
11. FE ATTACHES event handlers
    ‚Üì
12. FE INITIALIZES OData bindings
    ‚Üì
13. FE calls: GET /SalesOrder (fetch data)
    ‚Üì
14. FE RENDERS data in generated table
    ‚Üì
15. Browser displays the complete UI
```

**All of steps 8-14 happen AUTOMATICALLY - YOU wrote no code for this!**
